# AssetVault 更新日志 / Development Log

日期 / Date: 2026-05-26

## 概要 / Summary

本日志记录了 Asset Librarian 桌面应用本轮完成的修复、功能增强和 Windows 打包工作。重点包括模型导入、数据库加载、缩略图生成、GLTF/GLB 预览、右侧 3D 预览光照、HDR 环境支持，以及 Windows 程序封装。

This log records the fixes, feature improvements, and Windows packaging work completed for the Asset Librarian desktop application. The main focus was model import, database loading, thumbnail generation, GLTF/GLB preview, right-side 3D preview lighting, HDR environment support, and Windows app packaging.

## 运行环境与打包 / Runtime And Packaging

- 安装了 `ui/` 下的前端与 Electron 依赖。
- Installed UI and Electron dependencies under `ui/`.

- 将数据库访问从原生模块 `better-sqlite3` 改为项目已包含的纯 JS/WASM 方案 `sql.js`。
- Moved database access from native `better-sqlite3` to the bundled pure JS/WASM `sql.js` implementation.

- 移除了 Windows 打包时对 `node-gyp` 重建原生 SQLite 模块的依赖。
- Removed the Windows packaging dependency on rebuilding native SQLite modules with `node-gyp`.

- Added `signAndEditExecutable: false` in `electron-builder.yml` to avoid Windows symlink permission issues while building unsigned local installers.

- 已生成 Windows 安装包和免安装版本。
- Built both the Windows installer and the unpacked portable app.

## 已知说明 / Known Notes

- Windows 安装包未签名，系统可能提示“未知发布者”。
- The Windows installer is unsigned, so Windows may show an "Unknown publisher" warning.

- 旧缩略图可能需要在应用内点击 `Thumbnails` 重新生成。
- Existing old thumbnails may need regeneration with the app's `Thumbnails` button.

- 对于 `.gltf` 模型，`scene.bin` 和贴图等引用文件必须保持在 `.gltf` 文件中指定的相对位置。
- For `.gltf` models, referenced files such as `scene.bin` and textures must remain in the relative locations specified inside the `.gltf` file.

- 生产构建当前会提示 JS chunk 较大；不影响程序运行，后续可考虑代码拆分。
- The production build currently reports a large JavaScript chunk warning. This does not block the app, but code splitting can be considered later.

- `npm audit` 仍报告高危依赖告警；本轮未强制升级，避免引入破坏性变更。
- `npm audit` still reports high-severity dependency advisories. These were not force-upgraded in this pass to avoid breaking changes.
